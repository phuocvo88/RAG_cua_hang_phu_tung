## 📝 Maintenance Guidelines

### Adding New Milestones

When completing a new milestone:

1. **Create Milestone Report**
   ```bash
   # File naming convention
   MILESTONE_XX_SHORT_DESCRIPTION.md

   # Example
   MILESTONE_02_FRONTEND_MVP.md
   ```

2. **Required Sections**
   - Overview & objectives
   - Key achievements
   - Technical implementations
   - Test results
   - Files created/modified
   - Next steps
   - Lessons learned

3. **Update References**
   - Add entry to this README
   - Update PROJECT_TIMELINE.md progress
   - Link in ARCHITECTURE_OVERVIEW.md if relevant

### Updating Timeline

Monthly or after major phases:

1. Review [PROJECT_TIMELINE.md](./PROJECT_TIMELINE.md)
2. Update progress bars
3. Adjust target dates if needed
4. Add lessons learned to decision log

### Documentation Best Practices

- ✅ Use clear, descriptive headings
- ✅ Include visual diagrams where helpful
- ✅ Add code examples for technical sections
- ✅ Link related documents
- ✅ Keep metrics up-to-date
- ✅ Date all major updates
- ❌ Don't duplicate information across files
- ❌ Don't leave outdated information

## 🎓 Lessons Learned (Running Log)

### March 9, 2026
1. **Dynamic keyword extraction** is more reliable than hardcoded patterns
2. **Vietnamese stopword filtering** significantly improves search accuracy
3. **Google Gemini** provides better stability than Claude for this use case
4. **Local embeddings** eliminate API latency and reduce costs
5. **Comprehensive testing** catches issues early and builds confidence
